\section{Código}

Um \textbf{código} \emph{C} mapeia uma \textbf{mensagem} \emph{M} para um conjunto de \textbf{palavras código} \emph{W}, onde \emph{M} $\subset \alpha$ e \emph{W} $\subset \beta$, isto é, \emph{C} : $\alpha \rightarrow \beta$. Chamamos de $\alpha$ o \textbf{alfabeto de origem} e $\beta$ o \textbf{alfabeto de palavras código}. Os elementos dos alfabetos $\alpha$ e $\beta$ podem ter um comprimento fixo ou variável. Códigos nos quais os alfabetos possuem um comprimento fixo são chamados de \textbf{códigos de comprimento fixo}, enquanto os que possuem alfabetos de comprimento variáveis são chamados \textbf{códigos de comprimento váriavel}. Provavelmente o exemplo mais conhecido de código de comprimento fixo é o código ASCII, que mapeia 64 simbolos alfa-númericos (ou 256 em sua versão extendida) para palavras código de 8 bits. Todavia, a compressão de dados utiliza apenas códigos de comprimento variável, mas especificamente códigos que variam o comprimento de acordo com a probabilidade associada à mensagem.

Um código é \textbf{distinto} se pode ser representado como uma função \textbf{bijetora}, i.e, $\forall$ $m_1$, $m_2$ $\in$ $\alpha$, \emph{C($m_1$)} $\neq$ \emph{C($m_2$)}. Dado um código \emph{C}, com $\alpha$ := {$m_1, m_2,..., m_n$} e $\beta$ := {$w_1, w_2,...,w_n$}, chamamos \emph{C} \textbf{unicamente decodificável} quando \emph{C(S)} = $w_1w_2...w_n$ $\leftrightarrow$ \emph{$C^{-1}(W)$} = $m_1m_2...m_n$.
